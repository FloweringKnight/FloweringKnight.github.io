# 程序员眼中的古老宝藏：NetHack 游戏教程与体验

## 初识古老的游戏

作为一名Coder，我习惯了在代码的世界里逻辑清晰、步骤明确的生活。直到最近，我偶然接触到了 NetHack，这款诞生于 1987 年的经典 roguelike 游戏，彻底颠覆了我对游戏的认知。

NetHack 的背景源自早期游戏 Rogue，但它的复杂度和深度远超同期的任何游戏。这个游戏完全基于文本字符界面，没有华丽的画面，没有震撼的音效，只有一个用 ASCII 字符构成的地牢世界。但正是这种纯粹，让它成为了程序员心中的圣经。

游戏的剧情很简单：你需要从地牢的第一层出发，深入地下，寻找传说中的 Amulet of Yendor（勇者护身符），并将其献给神祇，最终成为半神。听起来不难？实际上，这是世界上最难的游戏之一。

## 第一次探索：从零开始

安装 NetHack 很简单，Linux 用户直接从软件源安装 `nethack` 或 `nethack-console`，Windows 用户去官网下载即可。第一次启动游戏时，系统会询问我是否要随机选择种族、职业、性别和阵营。

我想，作为新手，稳妥起见选择了 Valkyrie（女武神）这个角色。女武神被公认为最适合新手的职业，血厚装备好，还有抗寒的特性。种族选择人类，阵营选择守序。

进入游戏后，屏幕上出现了一个字符界面：
```
        ------
        |....|....|
        |..@.|....|
        |....+....|
        |...f.....|
        +---|-----|
```

`@` 是我，`f` 是我的宠物猫，`+` 是门，`-` 和 `|` 是墙壁。一切都用字符表示，简单而纯粹。

## 学习基本操作

游戏提示我使用 `h/j/k/l` 来移动，这是 Vim 风格的键位，作为程序员我感到无比亲切。`h` 向左，`j` 向下，`k` 向上，`l` 向右。如果需要斜向移动，还有 `y/u/b/n` 四个键。

我试着按了一下 `h`，角色向左移动了一格。很好。我又试着按 `i` 查看背包，屏幕列出我拥有的物品：
```
a - a +0 elven dagger
b - a +0 elven spear
c - a +0 elven short sword
d - a +0 dwarvish round shield
e - 2 +0 orcish helmets
f - 2 food rations
```

这些装备都是未鉴定的，我不知道它们的具体属性。但没关系，新手教程告诉我：先把装备穿上，观察 AC（护甲等级）变化。AC 越低越好。

我按 `w` 穿上了一件盔甲，AC 从 10 降到了 7，看来是件不错的装备。我又拿起武器按 `w` 装备上，准备迎接第一只怪物。

## 第一场战斗与死亡

向前移动了几个格子后，我遇到了一只 Jackal（豺狼）。它是游戏中最弱的怪物之一，但对于新手的我来说，这依然是一次真正的战斗。

我直接向豺狼的方向移动，角色自动发动攻击。几个回合后，豺狼倒下了。地面上出现了一个 `%` 符号，那是尸体。

新手教程告诉我可以吃尸体补充食物。我移动到尸体上按 `e` 进食，屏幕显示："This food really hit the spot!"（这食物真是美味！）

但就在这时，我犯了一个致命的错误。我又遇到了一只小妖精，它偷走了我的一些金币，然后传送离开了。我追着它跑，却忘记了最重要的原则：这不是 Quake，旁边有怪物并不意味着必须迅速行动。

因为一直按住方向键，我直接跑进了一个陷阱，中了毒。HP 快速下降，等我意识到要吃解毒药水时，已经来不及了。HP 归零，游戏结束。

屏幕上出现了一张由字符组成的墓碑：
```

                    |          |                 |
                    |          |                 |
                    |    RIP   |                |
                    |          |               |
                    |          |               |
    Redmage          |          |               |
```

游戏询问我是否要鉴定财产，我点击 yes，发现背包里其实有一瓶治疗药水，如果我及时使用就能活下来。这就是 Roguelike 的魅力——生命只有一次。

## 游戏体验：与现实的碰撞

相比我平时玩的其他游戏，NetHack 的体验完全不同。

现代游戏通常会引导你，告诉你该做什么，哪里可以走，哪里有宝藏。但 NetHack 完全不同，它从不提示你，一切都需要你自己探索和思考。

在《塞尔达传说》里，你看到一个解谜场景，通常只需要找到正确的按钮。在 NetHack 里，你面对同样的情况，可能需要搜索隐藏门、踢开箱子、或者变形为某种生物才能通过。

这种设计让我想起了编程。在编写代码时，没有编译器告诉你所有的错误在哪里，你需要自己理解逻辑、追踪 bug、调试问题。NetHack 就像是一个永不崩溃的复杂系统，需要你像调试程序一样调试游戏过程。

但正是这种挑战，让它如此令人上瘾。每一次死亡都不是因为游戏太不公平，而是因为我的决策失误。这种纯粹的个人责任感，是现代游戏很少提供的体验。

## 等待下一次冒险

经过几次死亡后，我开始掌握一些基本技巧。比如：
- 关闭自动拾取，只捡有用的东西
- 看到浮空眼，扔东西杀它，不要直接攻击
- 饥饿时优先吃怪物尸体，实在不行才祈祷
- 物品分三种状态：Blessed（受祝福）、Uncursed（未诅咒）、Cursed（被诅咒）

对于跃跃欲试的读者，我有几点建议：

1. **从战士型职业开始**：女武神、野蛮人、武士是最适合新手的。法师虽然有趣，但生存率低
2. **不要怕死**：NetHack 的乐趣在于学习和成长，死亡是学习的过程
3. **查阅攻略但不要依赖**：有些信息需要了解，但太多会失去探索的乐趣
4. **保持耐心**：这不是快速通关的游戏，慢慢来，先思考

NetHack 不仅仅是一个游戏，它是一个世界，一个充满了谜题、惊喜和挑战的世界。每一个字符、每一个怪物、每一个物品都有其背后的故事和用途。

作为一名程序员，我越来越喜欢这个游戏了。它让我想起了最原始的游戏魅力——没有华丽的包装，只有纯粹的游戏机制和无限的探索可能。这或许就是它能够传承三十多年的原因。

Best wishes,
希望你在 NetHack 的世界里，也能找到属于自己的冒险故事。
